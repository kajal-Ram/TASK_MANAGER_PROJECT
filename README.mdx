Task Manager Project

Overview
A Django REST API project where users can register, login using JWT tokens, and manage their personal tasks. Includes role-based access (Admin/User), filters, search, pagination, and task CRUD operations.

Tech Stack

Python 3.9+

Django 5.x

Django REST Framework

djangorestframework-simplejwt

django-filter

SQLite

Setup Instructions

Clone the repository:

git clone https://github.com/kajal-Ram/TASK_MANAGER_PROJECT.git
cd TASK_MANAGER_PROJECT


Create virtual environment and install dependencies:

python -m venv env
# Windows
env\Scripts\activate
# Linux/macOS
# source env/bin/activate
pip install -r requirements.txt




Apply migrations:

python manage.py makemigrations
python manage.py migrate


Create superuser (Admin):

python manage.py createsuperuser


Run development server:

python manage.py runserver





Roles & Permissions
Role	Permissions
Admin	Can view, create, update, delete any user's tasks
User	Can view, create, update, delete only their own tasks
API Endpoints
Authentication
Endpoint	Method	Description
/api/auth/register/	POST	Register a new user
/api/auth/login/	POST	Login and get JWT tokens
/api/auth/refresh/	POST	Refresh access token
Tasks
Endpoint	Method	Description	Request Body	Example Response
/api/tasks/	GET	List tasks (Admin: all, User: own)	None	[{"id":1,"title":"Finish Project","status":false,"owner":"user1"}]
/api/tasks/	POST	Create a new task	{"title":"Finish Project","description":"Complete API","due_date":"2025-12-15T12:00:00Z"}	{ "id":1, "title":"Finish Project", "description":"Complete API", "status":false, "owner":"user1" }
/api/tasks/<id>/	GET	Retrieve a task	None	{ "id":1, "title":"Finish Project", "status":false, "owner":"user1" }
/api/tasks/<id>/	PUT/PATCH	Update a task	JSON data	Updated task JSON
/api/tasks/<id>/	DELETE	Delete a task	None	204 No Content

Query Parameters: ?page=1, ?search=<keyword>, ?ordering=due_date, ?status=True
Headers (JWT Authentication):

Authorization: Bearer <access_token>
Content-Type: application/json
